Curso de Git & GitHub

Este é um curso introdutório sobre Git e GitHub, onde você aprenderá a usar essas ferramentas para controle de versão e colaboração em projetos de software.

O que é Git?

Git é um sistema de controle de versão distribuído que permite aos desenvolvedores acompanhar as mudanças em seus projetos de software ao longo do tempo. Ele facilita a colaboração entre equipes, permitindo que várias pessoas trabalhem no mesmo projeto simultaneamente.

O que é GitHub?

GitHub é uma plataforma de hospedagem de código-fonte que utiliza o Git para controle de versão. Ele oferece recursos adicionais, como gerenciamento de projetos, integração contínua e colaboração em equipe, tornando-se uma ferramenta popular para desenvolvedores em todo o mundo.

Principais recursos do GitHub:

  • Repositórios: Armazene e gerencie seu código-fonte em repositórios GitHub.
  • Branches: Crie ramificações para desenvolver novas funcionalidades ou corrigir bugs sem afetar o código principal.
  • Pull Requests: Proponha mudanças no código e colabore com outros desenvolvedores para revisar e mesclar essas mudanças.
  • Issues: Gerencie tarefas, bugs e solicitações de recursos usando o sistema de issues do GitHub.
  • GitHub Actions: Automatize fluxos de trabalho de desenvolvimento, como testes e implantação, usando GitHub Actions.

Principais comandos do Git:

  • git init: Inicializa um repositório Git.
  • git clone: Clona um repositório existente.
  • git add: Adiciona arquivos ao staging area.
  • git commit: Confirma as mudanças no repositório local.
  • git push: Envia mudanças para o repositório remoto.
  • git pull: Recupera e mescla mudanças do repositório remoto.
  • git fetch: Recupera mudanças do repositório remoto sem mesclá-las.
  • git branch: Gerencia ramificações no repositório.
  • git merge: Mescla ramificações no repositório.
  • git status: Exibe o status dos arquivos no repositório.
  • git log: Exibe o histórico de commits do repositório.
  • git checkout: Alterna entre ramificações ou restaura arquivos.
  • git diff: Exibe as diferenças entre commits, ramificações ou arquivos.
  • git restore: Restaura arquivos do staging area ou do repositório.
  • git tag: Marca pontos específicos na história do repositório como importantes.
  • git revert: Reverte um commit específico, criando um novo commit que desfaz as mudanças.
  • git reset: Desfaz commits ou mudanças no repositório.
  • git stash: Armazena temporariamente mudanças não confirmadas.

Com esses comandos e recursos, você estará bem equipado para começar a usar Git e GitHub em seus projetos de desenvolvimento de software. Lembre-se de praticar regularmente para se familiarizar com as ferramentas e melhorar suas habilidades de controle de versão e colaboração.
